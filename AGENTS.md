# Obsidian Writing Rules

## Default Destination

When the user says "옵시디언에 추가해줘", create a Markdown note under:

`+/`

The environment-specific vault path is stored outside the vault in:

`~/.codex/obsidian.json`

## Filename

Use a short, clear Korean filename based on the topic.

Never overwrite an existing note. If the filename exists, append ` (2)`, ` (3)`, etc.

## Note Style

- Start with `# 제목`.
- Keep notes concise but self-contained.
- Prefer short sections and bullets.
- Preserve useful code blocks.
- Do not add frontmatter unless explicitly requested.

## Scope

Add only the requested note content. Do not reorganize the vault unless explicitly requested.
