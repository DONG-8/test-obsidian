## 전체 흐름

이 발표는 Claude Code를 매일 쓰면서도 매 세션이 단절되고, 반복 작업을 매번 다시 지시해야 하며, 단일 에이전트가 모든 일을 떠안는 문제에서 출발합니다. 발표자는 이를 해결하기 위해 Claude를 “혼자 일하는 도구”가 아니라, 비서실장과 여러 부서로 이루어진 작은 회사처럼 운영하는 NaverMadCat 플랫폼을 만들었다고 설명합니다.

NaverMadCat은 역할, 부서, 에이전트, 훅, 세션 동기화, 전역 커맨드, 대시보드, 채용/해고/병합 흐름을 묶어 Claude Code 사용 경험을 조직 운영처럼 바꾸는 시스템입니다. 핵심은 어디에서 접속하든 같은 조직 구성과 맥락으로 이어서 일하고, 반복 작업은 라이프사이클 훅과 자동화 루프가 처리하게 만드는 것입니다.

## 키워드 정리

- NaverMadCat: Claude Code 위에서 여러 에이전트를 회사처럼 운영하는 플랫폼
- 비서실장 모델: Claude가 요청을 분류하고 적절한 부서/에이전트를 배정하는 구조
- 라이프사이클 훅: 세션 시작, 전처리, 후처리, 종료 시점에 반복 작업을 자동화하는 장치
- 세션 동기화: PC나 환경이 바뀌어도 같은 조직 상태로 이어서 일하게 만드는 방식
- 전역 커맨드와 대시보드: 운영 상태를 확인하고 제어하는 인터페이스
- HR 모델: 에이전트의 채용, 해고, 병합, 상태 추적을 조직 관리처럼 다루는 관점
